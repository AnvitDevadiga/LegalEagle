## рҹҰ… What is Legal Eagle?

Legal Eagle is a **production-grade offline RAG (Retrieval-Augmented Generation) system** built for Indian law. It indexes 620+ pages of legal documents вҖ” the Indian Constitution, IPC, and Bharatiya Nyaya Sanhita вҖ” into a local ChromaDB vector store and answers questions using a locally running **Llama 3** model via Ollama.

**No cloud. No API keys. No data leaves your machine.**

The CLI is inspired by modern coding terminals like Claude Code: color-coded role panels, streaming markdown answers, live spinners during retrieval, and syntax-highlighted output вҖ” all running entirely offline.

## рҹҡҖ Quick Start

### Prerequisites

- Python 3.11+
- [Ollama](https://ollama.ai) installed

### 1 вҖ” Clone & install

```bash
git clone https://github.com/AnvitDevadiga/LegalEagle.git
cd LegalEagle
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2 вҖ” Pull local models

```bash
ollama pull llama3
ollama pull nomic-embed-text
ollama serve                    # keep this terminal running
```

### 3 вҖ” Add your legal PDFs

```bash
# Drop your PDFs into the data/ folder
data/
в”ңв”Җв”Җ constitution.pdf
в”ңв”Җв”Җ ipc.pdf
в””в”Җв”Җ bns.pdf
```

### 4 вҖ” Build the vector store

```bash
python ingest.py            # one-time index (~2 min for 620 pages)
python ingest.py --rebuild  # nuke & reindex from scratch
```

### 5 вҖ” Launch

```bash
python app.py
```

## рҹҰ… What is Legal Eagle?

Legal Eagle is a **production-grade offline RAG (Retrieval-Augmented Generation) system** built for Indian law. It indexes 620+ pages of legal documents вҖ” the Indian Constitution, IPC, and Bharatiya Nyaya Sanhita вҖ” into a local ChromaDB vector store and answers questions using a locally running **Llama 3** model via Ollama.

**No cloud. No API keys. No data leaves your machine.**

The CLI is inspired by modern coding terminals like Claude Code: color-coded role panels, streaming markdown answers, live spinners during retrieval, and syntax-highlighted output вҖ” all running entirely offline.

## рҹӣЎпёҸ Anti-Hallucination Design

```
вң“  Answers strictly grounded in retrieved legal context only
вң“  Explicit refusal when information is absent from documents
вң“  MMR retrieval вҖ” diversified chunks, no repeated context bleed
вң“  Low temperature (0.1) вҖ” deterministic, no creative fabrication
вң“  Every answer cites source PDF В· chunk count В· latency
вң“  Safety Notice auto-injected for sensitive keywords
   (abuse В· assault В· self-harm В· violence В· threat)
```

## вңЁ Feature Highlights

**рҹ”ҙ Streaming responses** вҖ” answers render token-by-token via LangChain streaming. Start reading immediately.

**рҹҹЎ Claude Code-style terminal** вҖ” color-coded `YOU` (cyan) В· `LEGAL EAGLE` (green) В· `SYS` (amber) panels built with the `rich` library.

**рҹҹў MMR retrieval** вҖ” Maximum Marginal Relevance (`k=5`, `fetch_k=20`, `О»=0.5`) diversifies retrieved chunks so multi-section answers cite different Acts instead of repeating the same passage.

**рҹ”ө Singleton clients** вҖ” LLM and embedding clients are `@lru_cache` singletons. First query warms up; every subsequent one is hot with zero re-initialisation overhead.

**рҹҹЈ Idempotent ingest** вҖ” `python ingest.py` is a no-op if the vector store exists. Pass `--rebuild` to reindex. PDF loading runs in parallel via `ThreadPoolExecutor`, embeddings in batches of 64.

**рҹҹ  Safety-aware** вҖ” high-risk keywords trigger automatic appending of Indian emergency helplines: **112** В· **Kiran Mental Health 1800-599-0019** В· **NCW 181**.
