## What is Legal Eagle?

Legal Eagle is an **offline RAG system** that lets you query 620+ pages of Indian legal documents вҖ” the Constitution, IPC, and Bharatiya Nyaya Sanhita вҖ” using a locally running Llama 3 model. No internet calls, no telemetry, no third-party APIs.

The CLI is built in the spirit of modern coding terminals: role-coded panels, streaming markdown answers, syntax-highlighted code blocks, and a live spinner during retrieval.

## Quick Start

### Prerequisites

- Python 3.11+
- [Ollama](https://ollama.ai) installed and running

### 1. Clone & setup

```bash
git clone https://github.com/AnvitDevadiga/legal-eagle.git
cd legal-eagle
python3 -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Pull models

```bash
ollama pull llama3
ollama pull nomic-embed-text
ollama serve            # keep this terminal running
```

### 3. Build the knowledge base

```bash
# Place PDFs in data/ folder first
python ingest.py            # one-time index build (~2 min)
python ingest.py --rebuild  # delete & reindex from scratch
```

### 4. Launch

```bash
python app.py
```

## Anti-Hallucination Design

```
вң“  Answers strictly limited to retrieved legal context
вң“  Explicit refusal when information is not in the documents
вң“  MMR retrieval вҖ” diversified context, no duplicate chunks
вң“  Low-temperature LLM config (temp=0.1) вҖ” no creative fabrication
вң“  Source citation on every response (PDF В· chunk count В· latency)
вң“  Safety Notice auto-appended for sensitive queries (abuse, harassment, etc.)
```

---

## Key Features

**Streaming responses** вҖ” answers render token-by-token; start reading immediately, no waiting for full generation.

**Claude Code-style terminal** вҖ” color-coded `YOU` / `LEGAL EAGLE` / `SYS` panels, markdown body, syntax-highlighted code blocks, live spinner.

**MMR retrieval** вҖ” diversified context (`k=5`, `fetch_k=20`) so multi-section answers cite different Acts/Articles instead of duplicating the same chunk.

**Singleton LLM + embedding clients** вҖ” no per-query re-initialisation; warm-up happens at boot, every subsequent query is hot.

**Idempotent ingest** вҖ” re-running `python ingest.py` no-ops unless you pass `--rebuild`; PDF loading is parallelised (ThreadPoolExecutor) and embeddings are batched (64/call).

**Safety-aware** вҖ” keywords like `abuse`, `assault`, `self-harm` auto-append Indian emergency helplines (112 В· Kiran 1800-599-0019 В· NCW 181) to every answer.
